Intro to Methods
A. What is Methods?
- Collection of statements that are grouped together to perform a task
- Can be predefined automatically by C# or by the developers themselves
B. Advantages of using Methods
i. Code reusability
- Can be called anywhere on the program
ii. Easy testing
- Methods are easily tested, making debugging more simplified
iii. Modifiability
- The program does not get affected when changing methods, making easy modifications
iv. Input versatility
- Methods accept various inputs such as variables and values
C. Declaring methods
- Methods should be declared first before using them on the program
Method declaration must include;
a. Return type: what method will return
b. Method name: unique identifier for the method
c. Parameters: optional inputs for methods, defined by data types and variables
D. Calling Methods
- Methods can be called by their name and passing required arguments
Method Parameters
- Also called arguments, parameters are variables that accept values when passed into the
method they are being called
- Placed within a parentheses () after the method’s name
Multiple Parameters
- Can have more than one parameter
- Allows method to do more complex operations and multiple inputs
- Values are returned using the “return” statement, denoted by “return” statement
- Methods can be called with specific parameters, and they can also be assigned to variables,
as long as the variable can store the returned data type.
Optional and named arguments
A. Optional arguments
- Has a default value specified as optional parameters
- Default values can be overridden if there are some optional arguments that is defined
- Can be done by calling a method with a different set of parameter value
B. Named arguments
- Can be specified by matching the parameter’s name
- Allows programmers to disregard the order of parameters in a call statement for a method
- Uses the name of parameter followed by a ( : ) colon and the value
Method overloading
- Mechanism in C# that declares multiple instances of methods that has the same value
while having different parameters
- Data type of a parameter un a particular method that will only accept values corresponding
to the specified data type
- Parameters can be used in other data types for different instances
Recursion
- Repeated application of a procedure that reduces it to smaller, simpler version
- It applies or execute the procedures on itself and/or any of it’s given value, until procedure
cannot be applied no more
A. Recursive Methods
- Methods that call themselves
- Includes a number of statement of declaration and execution
- Exit condition is a conditional statement that contains the base case
- The recursive call is a code block after the exit condition
- Usually contains operations and/or tasks performed for recursive process and it’s values
- Used on certain tasks such as arithmetic applications or finding the value of a factorial.
